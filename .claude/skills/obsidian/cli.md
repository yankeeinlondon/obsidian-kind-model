# Obsidian CLI

Official command-line interface that drives a **running** Obsidian instance. Anything
you can do in the app can be scripted, including developer commands for reloading
plugins, capturing errors/screenshots, inspecting the DOM/CSS, and running arbitrary
JavaScript against the app object.

> Requires a recent Obsidian (CLI shipped in the 1.12 early-access line; needs a
> Catalyst license while in early access). Commands and syntax may still change.
> Reference: <https://obsidian.md/cli>

> **Binary name:** the installed client is **`obsidian-cli`** (verified:
> `/Applications/Obsidian.app/Contents/MacOS/obsidian-cli`). The app bundle also ships an
> `obsidian` binary — that's the Electron GUI launcher, *not* the CLI, so `obsidian <command>`
> will not work. Obsidian's own `--help` usage string still prints `obsidian <command>`; ignore
> that and use `obsidian-cli`. All examples below use `obsidian-cli`.

## Setup

1. **Update Obsidian** to a version that includes the CLI.
2. **Enable it:** Settings → General → **Command line interface** → toggle on.
3. **Register it:** click **Register CLI** and follow the prompts, then restart your terminal.

Registration per platform (the registered command is `obsidian-cli`):

- **macOS** — registers `obsidian-cli` (asks for admin via system dialog). The binary lives at
  `/Applications/Obsidian.app/Contents/MacOS/obsidian-cli`; reach it via the registered name or
  by adding that bundle dir to your `PATH`.
- **Windows** — installs an `Obsidian.com` terminal redirector beside `Obsidian.exe`
  (needed because Obsidian is a GUI app).
- **Linux** — copies the binary to `~/.local/bin/`; ensure `~/.local/bin` is on your `PATH`.

**The Obsidian app must be running** for commands to execute — the CLI is a client that
talks to the live app, not a headless renderer.

## Syntax conventions

- **Parameters** use `key=value`. Quote values with spaces: `name="My Note"`.
- **Flags** are bare words, no dashes: `silent`, `overwrite`, `total`.
- The **only** `--` flag is `--copy` (copy result to clipboard).

### Targeting files and vaults

| Param         | Meaning                                            |
| ------------- | -------------------------------------------------- |
| `file=<name>` | Resolves as a wikilink (note name only)            |
| `path=<path>` | Exact path from the vault root                     |
| `vault=<name>`| Choose the target vault when several are open      |

```bash
obsidian-cli vault="Notes" search query="meeting notes" format=json
```

## Vault / content commands

```bash
obsidian-cli read file="Note Name"            # print a note
obsidian-cli read                             # print the current (active) file
obsidian-cli create name="New" content="Text" template="Template" silent
obsidian-cli search query="kind/software" limit=10 format=json
obsidian-cli daily                            # open today's daily note
obsidian-cli daily:read                       # print today's daily note
obsidian-cli daily:append content="- new task"
obsidian-cli tasks [file]                     # list tasks in a file
obsidian-cli tags counts                      # all tags with frequency counts
obsidian-cli files sort=modified limit=20     # list files (sortable)
obsidian-cli unresolved                       # find unresolved links
obsidian-cli diff file="Note" from=1 to=3     # compare file versions
```

Common output modifiers: `format=json`, `--copy`, `silent` (don't open the file),
`total` (counts on list commands), `limit=N`.

## Plugin commands

```bash
obsidian-cli plugins                          # list installed plugins
obsidian-cli plugin:enable id=dataview
obsidian-cli plugin:disable id=calendar
obsidian-cli plugin:reload id=obsidian-kind-model   # hot-reload during development
```

## Developer commands

```bash
obsidian-cli devtools                         # open DevTools
obsidian-cli dev:errors                       # list recent JS errors
obsidian-cli dev:console level=error          # read console output
obsidian-cli dev:screenshot path=shot.png     # capture the app
obsidian-cli dev:dom selector=".workspace-leaf" text
obsidian-cli dev:css selector=".workspace-leaf" prop=background-color
obsidian-cli dev:mobile on                    # toggle mobile emulation
obsidian-cli eval code="app.vault.getFiles().length"
obsidian-cli eval code="app.workspace.getActiveFile()?.path"
```

### `eval` — the power tool (and footgun)

`eval` runs arbitrary JavaScript with full access to the Obsidian `app` object — the
same runtime your plugin uses. It can do anything the plugin API can, so it's a complete
backdoor into the vault. Only run code you've read and understand; never blind-execute
`eval` snippets generated by an AI without reviewing the JavaScript first.

## Shell completions

There is **no native completion generator** (no `obsidian-cli completion zsh`, no scripts
shipped in the app bundle). But the CLI exposes a hidden **`__completions`** command that
prints a JSON catalog of every command, its description, and its flags:

```bash
obsidian-cli __completions | jq 'keys'          # all command names
obsidian-cli __completions | jq '.search.flags'  # flags for one command
```

A zsh completion can be built from that catalog. Flag shape in the JSON:

- a flag with a `value` field → `key=value` style (offer `key=`)
- a flag without `value` → bare boolean flag (offer `key`)
- a `value` like `text|json` (pipe-delimited, no `<…>`) → an enum whose options can be completed

A working `_obsidian-cli` zsh function (caches the catalog under
`${XDG_CACHE_HOME:-~/.cache}/obsidian-cli/`, completes commands + flags + enum values, and
resolves the command even after a global `vault=…`) is installed at
`~/.zsh/completion/_obsidian-cli`. Install by dropping it in any dir on `$fpath` (before
`compinit`); refresh after upgrading Obsidian with `obsidian-cli-completion-refresh` (or delete
the cache file). Requires `jq` and Obsidian running the first time to build the cache.

## TUI mode

Running `obsidian-cli` with no command opens an interactive TUI with autocomplete and history
(readline-style shortcuts: `Ctrl+B/F` move, `Ctrl+A/E` line start/end, `Ctrl+R` history
search, `Ctrl+L` clear, `Ctrl+C/D` exit).

## Plugin dev loop (this repo)

Replaces the `~/obsidian` symlink + `.hotreload` dance for re-testing builds:

```bash
pnpm build                                # or `pnpm dev` watch mode
obsidian-cli plugin:reload id=obsidian-kind-model
obsidian-cli dev:errors                       # confirm clean load
obsidian-cli dev:screenshot path=/tmp/km.png  # visually verify a km block
```

`eval` is handy for poking at plugin state, e.g.:

```bash
obsidian-cli eval code="app.plugins.plugins['obsidian-kind-model'] ? 'loaded' : 'missing'"
obsidian-cli eval code="Object.keys(app.plugins.enabledPlugins)"
```
